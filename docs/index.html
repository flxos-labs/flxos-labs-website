FlxOS Documentation

Complete guide to installing, configuring, and developing with FlxOS.

Installation

FlxOS requires ESP-IDF v5.x and a compatible ESP32 development board. Follow these steps to get started.

Clone Repository 
git clone --recursive https://github.com/flxos-labs/flxos.git
cd flxos

Quick Start

Get FlxOS running on your ESP32 in just a few minutes.

1

Set up ESP-IDF environment

. $HOME/esp/esp-idf/export.sh
2

Configure target platform

idf.py set-target esp32
# Or for ESP32-S3: idf.py set-target esp32s3
3

Configure project settings

idf.py menuconfig

Configure display driver, Wi-Fi, and other options in the menu.

4

Build and flash

idf.py build flash monitor

Prerequisites

Required Software

  • ESP-IDF v5.x - Installation Guide
  • CMake 3.16+ - Build system generator
  • Ninja - Fast build tool
  • Python 3.8+ - Required for ESP-IDF tools

Supported Hardware

  • ESP32 (all variants)
  • ESP32-S3
  • Compatible TFT display (recommended: ILI9341, ST7789)
  • Touch controller (optional: XPT2046, FT6336)

Building

Important: Always run idf.py set-target before the first build to configure for your specific ESP32 variant.

Build Commands

# Full build
idf.py build

# Clean build
idf.py fullclean build

# Build and flash
idf.py build flash

# Build, flash, and monitor
idf.py build flash monitor

Configuration

FlxOS uses the ESP-IDF menuconfig system for configuration.

idf.py menuconfig

Key Configuration Options

  • Display Driver: Configure your TFT display type and pins
  • Touch Controller: Set up touch input device
  • Wi-Fi: Default SSID and password
  • Performance: CPU frequency, memory allocation

Creating Apps

FlxOS apps follow a simple pattern. Here's a minimal example:

MyApp.hpp 
#pragma once
#include "App.hpp"

class MyApp : public App {
public:
    MyApp() : App("MyApp", "My Application") {}
    
    void onCreate() override {
        // Initialize your app UI
        lv_obj_t* label = lv_label_create(getContainer());
        lv_label_set_text(label, "Hello, FlxOS!");
    }
    
    void onResume() override {
        // App brought to foreground
    }
    
    void onPause() override {
        // App sent to background
    }
};

Troubleshooting

Build fails with "CMake Error"

Ensure you've run idf.py set-target esp32 before building. Also verify ESP-IDF is properly installed and sourced.

Display shows garbage or doesn't work

Check your display configuration in menuconfig. Verify pin connections match your hardware setup.

Touch not responding

Ensure touch controller is enabled in menuconfig and pins are correctly configured. Some displays require calibration.

Wi-Fi won't connect

Verify SSID and password in menuconfig. Check that your router supports 2.4GHz (ESP32 doesn't support 5GHz).
